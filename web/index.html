O que e este diagnostico?

Ferramenta para validar a rede de um estabelecimento antes ou durante a operacao do Vuca. Roda checks que simulam o que a aplicacao real faz e relatam — em linguagem comum — onde estao os problemas e como corrigir.

O foco e' detectar problemas que sao imperceptiveis no dia-a-dia: DNS lento que faz o tablet abrir devagar, certificado expirando, antivirus interceptando HTTPS, MTU mal configurado causando upload lento, perda intermitente de pacote, e por ai vai.

Como rodar uma validacao

  1. Preencha os campos da identificacao (instancia da loja, opcionalmente URL do VucaLocal).
  2. Configure RabbitMQ se for um cliente que usa impressao via nuvem (host e portas — geralmente vucaprint-3.vucasolution.com.br e portas 5672, 15672).
  3. Adicione cada impressora da loja com nome, IP e porta (normalmente 9100).
  4. Adicione portas customizadas se houver servicos extras a validar (TEF, banco, API interna).
  5. Clique em "Executar diagnostico".
  6. Acompanhe os cards aparecendo ao vivo — cada teste mostra seu progresso em tempo real.
  7. Ao final, clique em "Baixar HTML" e mande o arquivo gerado pelo WhatsApp/email para o suporte.

Dica: os dados que voce preenche ficam salvos no navegador. Na proxima vez que rodar, o formulario ja vem pronto.

Significado de cada campo

Instancia
E' o "apelido" da loja no Vuca, sem o dominio. Por exemplo, se a loja e' lifeboxburger.vucasolution.com.br, voce digita apenas lifeboxburger. O diagnostico monta a URL completa automaticamente. Campo opcional — se nao informar, o teste roda em modo "validacao parcial" (so' rede interna).
URL do VucaLocal
URL local da aplicacao VucaLocal instalada na maquina do cliente. Geralmente http://localhost:8080. Nao e' uma URL publica — se voce colocar um endereco da nuvem aqui, o teste vai reclamar. Deixe em branco se o cliente nao usa VucaLocal.
RabbitMQ — Host
Servidor de filas usado para enviar jobs de impressao via nuvem. Geralmente algo como vucaprint-3.vucasolution.com.br.
RabbitMQ — Portas
Portas TCP a testar separadas por virgula. Padrao: 5672 (protocolo AMQP) e 15672 (painel de administracao). A 5672 e' a que importa para impressao funcionar.
Impressoras
Lista das impressoras termicas da loja. Para cada uma: nome amigavel (ex: "Cozinha", "Caixa"), IP na rede local e porta (padrao 9100). Suporta IP fixo, DHCP (se souber o IP atual) e impressoras Wi-Fi. Nao suporta impressoras conectadas por cabo USB direto (essas nao tem como ser testadas via rede).
Portas customizadas
Testes adicionais para qualquer servico que voce queira validar: maquininha TEF (porta 4096 ou similar), Postgres interno, API de integracao, etc. Cada entrada e' apenas Nome + Host + Porta.

O que cada categoria de teste valida

Sistema

Informacoes do PC onde o diagnostico esta rodando. Apenas contexto — nunca falha.

Rede Local

A "saude" da rede onde o PC esta conectado. Inclui: lista de interfaces (placas de rede), latencia ate o roteador, descoberta de MTU, IP publico, velocidade de banda (download de 10MB), e teste de latencia estendido (50 amostras em 30s para pegar instabilidade).

Ambiente

Interferencias no PC que podem afetar todos os outros testes silenciosamente: proxy/VPN configurados, drift do relogio (diferenca da hora local vs internet), perfil de energia do Windows (economia pode causar drops de Wi-Fi).

Conectividade

Validacoes ate a nuvem Vuca: resolucao DNS multi-servidor (sistema vs 1.1.1.1 vs 8.8.8.8), HTTPS direto na instancia, latencia ate o servidor, tempos por fase (DNS/TCP/TLS/TTFB), validacao do certificado SSL (incluindo deteccao de antivirus interceptando), e consistencia (3 requisicoes seguidas).

RabbitMQ

Acesso ao servidor de filas: porta TCP aberta + handshake AMQP completo (garante que e' realmente RabbitMQ, nao algum outro servico ocupando a porta).

VucaLocal

Se a URL local foi informada, testa se o servico VucaLocal esta respondendo e qual status retorna.

Impressoras

Para cada impressora: teste TCP basico + comando ESC/POS de status (consegue identificar tampa aberta, papel acabando, impressora travada). Nao imprime nada.

Portas customizadas

Teste TCP direto em cada host:porta informado.

Como interpretar os resultados

  • OK — tudo funcionando normalmente. Nada a fazer.
  • Atencao — o teste passou mas com alguma observacao. Funciona agora mas pode dar problema. Revise.
  • Falha — algo nao esta funcionando como deveria. Precisa ser resolvido antes da operacao.
  • Info — apenas informacao contextual, nao indica problema nem sucesso.

Cada card mostra a mensagem tecnica e, ao expandir, voce ve as etapas que o teste executou em ordem. No relatorio baixado, cada card tem ainda "O que isso significa", "Causas mais comuns" e "O que fazer" em linguagem comum.

Banner vermelho de "Diagnostico interrompido"

Se o teste de HTTPS falhar (instancia errada ou inexistente), os testes subsequentes (latencia, RabbitMQ, VucaLocal, impressoras) sao pulados de proposito. Validar impressora se a instancia nao existe e' perda de tempo e gera falsos positivos no relatorio.

O banner vermelho aparece com o motivo e a recomendacao. Corrija a instancia e clique em "Executar diagnostico" de novo.

Sobre o relatorio baixado

O arquivo HTML gerado e' auto-contido: nao depende de internet, fonts externas ou imagens. Da' para mandar por WhatsApp/email tranquilamente e quem receber abre direto no navegador (qualquer um).

Cada card no relatorio traz, alem da mensagem tecnica, secoes de "O que esse teste verifica", "Resultado encontrado", "O que isso significa", e quando houver problema, "Causas mais comuns" e "O que fazer" com instrucoes especificas (ex: "vai em Configuracoes → Hora e Idioma → Sincronizar agora").

O relatorio tambem e' amigavel para impressao — abre todos os detalhes e remove sombras quando voce manda imprimir.

Perguntas frequentes

"DNS deu Atencao por wildcard — isso e' problema?"

Nao necessariamente. O dominio vucasolution.com.br esta atras do Cloudflare, que usa wildcard DNS — qualquer nome resolve para o mesmo IP. Isso significa que o DNS sozinho nao confirma a instancia, so' o HTTPS confirma. Se o HTTPS deu OK abaixo do DNS, esta tudo bem.

"URL do VucaLocal aparece como 'nao informada'"

E' opcional — so' encha se o cliente realmente usa VucaLocal instalado na maquina. Se nao usa, deixa em branco que o teste marca como info e segue.

"Impressora ESC/POS deu Atencao mas o TCP deu OK — entendi errado?"

Algumas impressoras (especialmente modelos antigos, USB-emuladas ou genericas) aceitam conexao TCP mas nao respondem ao comando padrao de status. Se a impressora estiver imprimindo normalmente apesar do aviso, ignore. Se nao estiver, e' candidata a reinicio fisico.

"O teste de latencia estendida demora 30 segundos, e' normal?"

Sim. Esse teste faz 50 amostras espalhadas em 30s para detectar instabilidade que testes curtos nao pegam. Durante esse tempo, outros cards continuam aparecendo em paralelo gracas ao streaming ao vivo.

"Posso rodar de outro PC que nao seja o do caixa?"

Pode, desde que esteja na mesma rede que o PC do caixa. O diagnostico testa a rede atual em que esta rodando. Se voce rodar do seu notebook conectado no Wi-Fi do cliente, voce esta testando esse Wi-Fi. Se rodar com 4G/dados moveis, voce esta testando a rede do seu celular, nao a do cliente.
